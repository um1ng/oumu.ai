# Oumu.ai - 阴影学习应用

一个基于Web的阴影学习（shadowing）应用，支持音频转录、文本处理和语言学习功能。

## 🚀 功能特点

- **音频转录**: 使用 Groq Whisper-large-v3-turbo API 进行高质量语音识别
- **文本处理**: 通过 Groq Moonshot 模型进行文本标准化和注释
- **精确同步**: 段落级别的时间戳同步，支持A-B循环功能
- **本地存储**: 所有用户数据存储在本地 IndexedDB 中，保护隐私
- **统一主题**: 设计令牌驱动的浅色/暗色外观，包含语义状态色
- **渐进式Web应用**: 支持离线使用和安装到桌面
- **健康检查**: 全面的系统诊断和监控功能，确保转录服务稳定运行

## 🛠️ 技术栈

- **前端**: Next.js 15, React 19, TypeScript
- **UI 组件**: shadcn/ui, Radix UI, Tailwind CSS
- **主题系统**: 自定义设计令牌 + next-themes（浅色/暗色/系统）
- **数据库**: IndexedDB (Dexie)
- **AI 服务**:
  - Groq Whisper-large-v3-turbo (音频转录)
  - Groq openai/gpt-oss-20b (文本处理)
- **样式**: Tailwind CSS
- **开发工具**: Biome.js, Jest
- **通知**: Sonner (Toast)

### 数据流架构

```
用户上传音频 → IndexedDB存储 → 分片处理 → Groq转录 → AI模型后处理 → 字幕同步播放
     ↓              ↓           ↓          ↓           ↓              ↓
  文件管理      本地持久化     并发控制    语音识别   多AI服务处理      跟读练习
```

### AI 模型配置
- **转录模型**: Groq Whisper-large-v3-turbo (音频转文本)
- **文本处理模型**: Groq openai/gpt-oss-20b (文本标准化和注释)


## 🔍 健康检查系统

Oumu.ai 内置了全面的健康检查系统，帮助监控和诊断转录服务的状态，确保系统稳定运行。

### 检查项目

健康检查系统包含以下五个核心检查类别：

1. **API连通性检查**
   - 验证 Groq API 认证状态和服务可用性
   - 检查 Whisper 模型可访问性
   - 监控 API 响应时间和配额使用情况

2. **错误处理验证**
   - 模拟各种错误场景（认证失败、网络错误、超时等）
   - 验证错误信息的清晰度和用户友好性
   - 检查恢复机制和重试策略

3. **性能基准测试**
   - 测量 API 响应时间和处理速度
   - 监控内存使用和 CPU 占用率
   - 验证并发处理能力

4. **用户体验验证**
   - 检查界面响应性和加载时间
   - 验证移动端适配和可访问性
   - 评估操作流程的直观性

5. **安全合规检查**
   - 验证本地数据存储安全性
   - 检查 API 密钥保护措施
   - 确认数据传输加密状态

### 使用方法

1. **访问健康检查**：通过主导航菜单中的"健康检查"进入
2. **运行检查**：点击"快速检查"进行基础检查，或选择"完整检查"进行全面诊断
3. **查看结果**：实时监控检查进度，查看详细报告和改进建议
4. **配置设置**：自定义检查频率、通知方式和检查项目

### 报告功能

- **健康评分**：0-100分的系统健康度综合评分
- **详细报告**：包含检查结果、问题分析和修复建议
- **历史趋势**：跟踪系统健康状况的变化趋势
- **数据导出**：支持 JSON、PDF、CSV 格式的报告导出

### 配置选项

- **自动检查**：设置定期健康检查（建议每日一次）
- **检查类别**：选择要执行的检查项目
- **通知设置**：配置问题告警和通知方式
- **数据保留**：设置历史报告的保留期限

### 性能指标

健康检查系统确保以下性能标准：
- **检查完成时间**：单类别检查 < 30秒，完整检查 < 2分钟
- **响应时间**：API响应 < 2秒，UI交互 < 100ms
- **并发处理**：支持多检查并行执行
- **内存效率**：检查过程内存增长 < 50MB

### 测试覆盖

健康检查系统包含全面的测试套件：
- **单元测试**：数据库操作、报告生成、核心逻辑
- **性能测试**：执行时间、内存使用、并发性能
- **端到端测试**：完整工作流程、错误处理、数据持久性
- **可访问性测试**：键盘导航、屏幕阅读器支持、ARIA合规性
